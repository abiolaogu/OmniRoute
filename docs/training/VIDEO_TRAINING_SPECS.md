# OmniRoute Video Training Specifications
## Production Guidelines for Training Videos

---

## Overview

This document provides technical specifications and guidelines for creating OmniRoute training videos. All videos should follow these standards for consistency and quality.

---

## Video Categories

| Category | Purpose | Typical Length | Style |
|----------|---------|----------------|-------|
| **Getting Started** | Onboarding new users | 3-5 minutes | Engaging, welcoming |
| **Feature Tutorials** | Demonstrate specific features | 5-12 minutes | Step-by-step |
| **Deep Dives** | Advanced topic exploration | 10-20 minutes | Comprehensive |
| **Quick Tips** | Single-feature explainers | 1-3 minutes | Concise, focused |
| **Webinar Recordings** | Training sessions | 30-60 minutes | Conversational |

---

## Technical Specifications

### Video Quality
| Aspect | Specification |
|--------|---------------|
| Resolution | 1920x1080 (1080p) minimum |
| Frame Rate | 30 fps |
| Aspect Ratio | 16:9 |
| Format | MP4 (H.264 codec) |
| Bitrate | 8-12 Mbps |

### Audio Quality
| Aspect | Specification |
|--------|---------------|
| Sample Rate | 48 kHz |
| Bitrate | 320 kbps |
| Channels | Stereo |
| Format | AAC |
| Noise Floor | Below -50 dB |

### Screen Recording
| Aspect | Specification |
|--------|---------------|
| Cursor | Highlighted with yellow circle |
| Clicks | Visual click indicator |
| Zoom | Use zoom for small UI elements |
| Resolution | Record at 4K, export at 1080p |

## Branding Guidelines

### Visual Identity
- **Primary Color**: #2563EB (OmniRoute Blue)
- **Text Color**: #1F2937 (Dark Gray)
- **Background**: Clean white or light gray
- **Font**: Inter (UI text), Outfit (headings)

### Required Elements
1. **Intro** (5 seconds)
   - OmniRoute logo animation
   - Video title
   - Background music fade-in

2. **Outro** (10 seconds)
   - Subscribe/follow CTA
   - Next video suggestion
   - OmniRoute logo
   - Contact information

3. **Lower Third**
   - Presenter name
   - Title/role
   - Topic being discussed

### Thumbnail Requirements
- 1280x720 pixels
- Include: Topic text, relevant screenshot/icon
- Use consistent color scheme
- Face of presenter (if applicable)
- Episode/lesson number

## Content Structure

### Getting Started Videos (3-5 min)
```
0:00 - 0:10  |  Intro animation
0:10 - 0:30  |  Quick welcome, topic introduction
0:30 - 3:30  |  Main content (3-4 key points)
3:30 - 4:00  |  Summary and next steps
4:00 - 4:30  |  Outro with CTA
```

### Feature Tutorials (5-12 min)
```
0:00 - 0:10  |  Intro animation
0:10 - 0:45  |  Problem statement / use case
0:45 - 9:00  |  Step-by-step walkthrough
9:00 - 10:00 |  Tips and best practices
10:00 - 10:30|  Summary
10:30 - 11:00|  Outro with CTA
```

### Deep Dive Sessions (10-20 min)
```
0:00 - 0:10  |  Intro animation
0:10 - 1:00  |  Topic overview and learning objectives
1:00 - 15:00 |  Comprehensive walkthrough with examples
15:00 - 17:00|  Advanced tips and edge cases
17:00 - 18:00|  Q&A teaser / common questions
18:00 - 18:30|  Summary
18:30 - 19:00|  Outro with CTA
```

## Production Checklist

### Pre-Production
- [ ] Script written and reviewed
- [ ] Screen workflow rehearsed
- [ ] Test data prepared in demo environment
- [ ] Audio check completed
- [ ] Recording environment quiet

### Recording
- [ ] Use demo account (never real customer data)
- [ ] Record in 4K if possible
- [ ] Capture both screen and audio
- [ ] Pause between steps for editing
- [ ] Record multiple takes of complex sections

### Post-Production
- [ ] Edit for pacing (remove dead air)
- [ ] Add zoom effects for small UI
- [ ] Insert callouts and annotations
- [ ] Add background music (subtle)
- [ ] Color correct if needed
- [ ] Add captions/subtitles
- [ ] Export in correct format

### Quality Assurance
- [ ] Review for accuracy
- [ ] Check audio levels
- [ ] Verify all links/resources mentioned
- [ ] Test on multiple devices
- [ ] Get stakeholder approval

---

## Accessibility Requirements

### Captions/Subtitles
- All videos must include captions
- Captions should be timed accurately
- Use proper punctuation
- Include speaker identification if multiple speakers

### Audio Descriptions
- Describe on-screen actions for screen readers
- Provide alternative format documentation

### Color Contrast
- Ensure text is readable (4.5:1 contrast ratio)
- Don't rely on color alone to convey information
