## Contributing to Documentation

1. All documentation follows Markdown formatting
2. Screenshots should be placed in `/docs/images/`
3. Use consistent terminology from the glossary
4. Technical docs must include code examples
5. User manuals must include step-by-step guides

For documentation standards, see [DOCUMENTATION_STANDARDS.md](DOCUMENTATION_STANDARDS.md).
